# Azerbaijan eTender Search — GitHub Pages

Searches and exports data from [etender.gov.az](https://etender.gov.az).

**Architecture:** GitHub Actions scrapes server-side (no CORS issues) → saves `data/tenders.json` → GitHub Pages site reads it.

---

## Repo structure

```
├── index.html                    ← the website (GitHub Pages)
├── scraper.py                    ← Python scraper (runs in GitHub Actions)
├── config.json                   ← your settings (keyword, URL pattern, selectors)
├── data/
│   └── tenders.json              ← auto-generated by the scraper
└── .github/
    └── workflows/
        └── scrape.yml            ← scheduled/manual GitHub Actions workflow
```

---

## Step 1 — Find the real search URL

This is the only part that requires manual inspection:

1. Open **https://etender.gov.az** in Chrome
2. Press **F12** → **Network** tab → check **XHR** / **Fetch**
3. Search for something on the site
4. Find the network request that returns tender listings
5. Click it → copy the **Request URL**

Example of what you might see:
```
https://etender.gov.az/tender/search?keyword=tikinti&startDate=2025-01-01&endDate=2025-12-31&page=1
```

---

## Step 2 — Update `config.json`

Replace `search_url` with the real URL pattern, using these placeholders:

| Placeholder | Replaced with |
|---|---|
| `{KEYWORD}` | Your search term |
| `{DATE_FROM}` | Start date (YYYY-MM-DD) |
| `{DATE_TO}` | End date (YYYY-MM-DD) |
| `{PAGE}` | Page number |

Also inspect one tender row in DevTools (right-click → Inspect) and note the CSS class names, then update `row_selectors` and `field_map`.

---

## Step 3 — Set up the GitHub repo

```bash
# Create a new public repo on github.com, then:
git clone https://github.com/YOUR-USERNAME/etender-search
cd etender-search
mkdir -p data .github/workflows
cp index.html config.json scraper.py .
cp scrape.yml .github/workflows/
git add .
git commit -m "initial"
git push
```

Then go to **Settings → Pages → Deploy from branch → main / (root)**.

---

## Step 4 — Run the scraper

### Manual run (first time or on demand)
1. Go to your repo on GitHub
2. Click **Actions** → **Scrape eTender**
3. Click **Run workflow**
4. Optionally enter a keyword and dates
5. Click **Run workflow** (green button)

The scraper will:
- Fetch pages from etender.gov.az
- Parse tender rows from the HTML
- Commit `data/tenders.json` to the repo
- GitHub Pages will serve the updated data within ~1 minute

### Automatic daily run
The workflow runs every day at **08:00 Baku time** (configured in `scrape.yml`).

---

## Step 5 — Use the site

Visit `https://YOUR-USERNAME.github.io/etender-search/`

Features:
- Filter by keyword, date range, status (client-side, instant)
- Sort by any column
- Export filtered results as **CSV** or **JSON**

## Troubleshooting

| Problem | Fix |
|---|---|
| Scraper gets 0 results | The `search_url` or `row_selectors` in config.json are wrong. Inspect the real site and update them. |
| `data/tenders.json` not found | Run the workflow at least once manually |
| Dates are empty | Update `field_map` with the correct column index or CSS selector for date cells |
| Titles show gibberish | The site may require a session/cookie — try adding a `session.get(BASE_URL)` warm-up call in scraper.py |
